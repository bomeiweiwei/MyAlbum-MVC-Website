4. 請設計一個跟更新相關的介面的Repository，名稱是：I<<類別名稱>>UpdateRepository

5. 介面的Repository一定有2個方法：
方法1名稱：Update<<類別名稱>>Async
方法2名稱：Update<<類別名稱>>ActiveAsync

6. 方法1設定為：
6-1. 方法一定會帶：
6-1-1. Dto類別：
6-1-1-1. 名稱：<<類別名稱>>UpdateDto
6-1-1-2. 屬性：除了類別屬性的 ReleaseTimeUtc、CreatedAtUtc、CreatedBy，全都加到<<類別名稱>>UpdateDto裡
6-1-1-3. 指定的日期欄位：UpdatedAtUtc
6-1-1-4. 如果指定的日期欄位有設定則給預設值：DateTime.UtcNow
6-1-1-5. 指定的檢查欄位：資料庫的表格的PK、AlbumPhotoId、MemberId，在更新前須透過檢查欄位確認資料存在
6-1-1-6. 遇到 Status關鍵字 請設定為：public Status Status { get; set; };，它在系統裡是Enum
6-1-2. CancellationToken ct = default

6-2. 方法預設會回傳布林值，更新成功回傳true

6-3. 此功能會更新的欄位：
Comment
IsChanged
Status
UpdatedBy
UpdatedAtUtc

7-1-1-3. 指定的日期欄位：UpdatedAtUtc

7-1-2. CancellationToken ct = default

8. 請設計一個類別實作介面的Repository，名稱是：<<類別名稱>>UpdateRepository
8-1. 假設類別名稱是：Sample，此 SampleUpdateRepository 初始建構子一律照以下模式做注入設定：
private readonly IAlbumDbContextFactory _factory;
public SampleUpdateRepository(IAlbumDbContextFactory factory) => _factory = factory;

8-2. 方法一律public async

8-3. 方法初始一律照以下模式設定：
8-3-1. 設定result：
bool result = false;
8-3-2. 設定交易區塊：
using var ctx = _factory.Create(ConnectionMode.Master);
var db = ctx.AsDbContext<MyAlbumContext>();
var strategy = db.Database.CreateExecutionStrategy();

/* 資料檢查區塊 */
/* 額外確認 */

await strategy.ExecuteAsync(async () =>
{
    await using var tx = await db.Database.BeginTransactionAsync(ct);
    try
    {
        /* 設定資料區塊 */
		
        await db.SaveChangesAsync(ct);

        await tx.CommitAsync(ct);

        result = true;
    }
    catch
    {
        await tx.RollbackAsync(ct);
        throw;
    }
});
8-3-3. 在上述資料檢查區塊需要先確認資料存在，使用到的Key為<<指定的檢查欄位>>：
var data = await db.<<資料庫的表格名稱>>s.FirstOrDefaultAsync(m => m.屬性 == dto.屬性);
if (data == null)
    throw new InvalidOperationException("找不到指定的資料。");
8-3-4. 在上述額外確認區塊不需要有程式，留著註解就好
8-3-4. 在上述設定資料區塊設定要更新的資料欄位，欄位參考<<此功能會更新的欄位>>，例：
 data.屬性 = dto.屬性;
 data.Status = (byte)dto.Status;
 
8-4. 交易區塊 後回傳result：
 return result;
